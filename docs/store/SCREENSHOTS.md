# 商店截图规范

在 **真机或模拟器** 使用生产 API 或稳定 staging，展示有内容的 Feed（可先 `ENABLE_DEMO_SEED=true` 灌数）。

## 必拍画面（6–8 张叙事顺序）

1. **Feed 首页** — 卡片含标题、来源、摘要、时间  
2. **文章详情** — 摘要 +「阅读原文」外链  
3. **话题订阅列表**  
4. **话题 Feed** — 某话题下的文章流  
5. **全局搜索** — 文章 / 话题结果  
6. **升级页** — Premium 权益说明（可 Sandbox 未购买状态）  
7. **设置** — 账号、通知、隐私政策链接  
8. **（可选）每日简报 / 推送** — 若 Phase 4 已启用

## 尺寸参考

### iOS (App Store Connect)

| 设备 | 分辨率 (px) | 必需 |
|------|-------------|------|
| iPhone 6.7" | 1290 × 2796 | 是 |
| iPhone 6.5" | 1242 × 2688 | 是 |
| iPhone 5.5" | 1242 × 2208 | 可选 |
| iPad Pro 12.9" | 2048 × 2732 | 若支持 iPad |

### Android (Play Console)

| 类型 | 尺寸 (px) |
|------|-----------|
| Phone | 1080 × 1920 至 1440 × 2560（至少 2 张） |
| 7" Tablet | 1200 × 1920（若支持平板） |
| Feature Graphic | 1024 × 500（Play 必填横幅） |

## 制作提示

- 使用浅色/深色各一套可提升通过率（可选）  
- 状态栏时间统一为 9:41 等常见样机惯例  
- 不要出现 debug 横幅、`DEV` 购买按钮截图（用 release 或隐藏 dev 入口）  
- 文案与 [listing.en.md](./listing.en.md) 卖点一致
