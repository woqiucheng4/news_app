## 1. 产品概述

### 1.1 产品定位

NewsFlow 是一款 AI 驱动的全源信息聚合应用，通过 AI 摘要技术帮助用户在 5 分钟内掌握全网最新信息。

**一句话定位：** 用户只需订阅感兴趣的话题，就能高效获取全网最新信息的 AI 摘要——无需自己逐个网站浏览。

### 1.2 目标市场

| 维度 | 描述 |
|------|------|
| 地理市场 | 先海外（英语市场），后国内 |
| 目标用户 | 信息密集型工作者、知识工作者、投资者 |
| 市场规模 | 全球新闻聚合应用市场，年增长率 8-12% |

### 1.3 竞争定位

| 维度 | Google News | Feedly | Inoreader | NewsFlow |
|------|-------------|--------|-----------|----------|
| AI 摘要 | 无 | 基础 | 非核心 | **核心价值** |
| 信息源广度 | 高 | 用户驱动 | 用户驱动 | **RSS + 爬虫混合** |
| 复杂度 | 低 | 中 | 高 | **低（Notion 风格）** |
| 价格 | 免费 | $6/月 | $5/月 | **免费增值** |

**差异化优势：** AI-first 摘要 + 简洁 UX，介于 Google News（无 AI 摘要）和 Inoreader（AI 非核心）之间。

### 3.2 功能详细规格

#### F-01: 内容采集系统

**描述：** 从多个信息源自动采集内容

**功能点：**
- RSS 源自动解析和抓取
- 用户自定义 RSS 订阅
- 用户自定义网页 URL 监控
- 热点话题自动抓取（HN、Reddit、GitHub、微博、知乎等）
- 采集频率：15-60 分钟
- 失效源自动告警

**验收标准：**
- [ ] 系统可自动解析预置 RSS 源
- [ ] 用户可添加自定义 RSS 地址
- [ ] 用户可添加任意网页 URL
- [ ] 系统每 15-30 分钟更新热点数据
- [ ] RSS 源失效时产生告警

---

#### F-02: AI 摘要生成

**描述：** 为每篇文章生成简洁、准确的 AI 摘要

**功能点：**
- 自动摘要生成（2-3 句话，50-100 字）
- 多语言支持（与原文语言一致）
- 摘要缓存（避免重复调用）
- 成本控制（日预算限制）
- 质量评估和监控

**验收标准：**
- [ ] 每篇文章自动生成 AI 摘要
- [ ] 摘要长度 50-100 字
- [ ] 不复制原文句子
- [ ] 月度 AI 成本 < $50
- [ ] 摘要生成延迟 < 5 秒

**Prompt 设计：**
```
你是一位专业的新闻摘要编辑。

要求：
1. 长度：2-3 句话，总计 50-100 字
2. 内容：仅包含核心事实，不添加评论或推测
3. 语言：与原文语言一致

禁止：
- 不要复制原文句子
- 不要添加引导词（"根据报道"、"据悉"）
- 不要猜测未明确说明的信息
```

---

#### F-03: 内容去重

**描述：** 自动检测并合并重复内容

**功能点：**
- URL 去重（第一层）
- 标题相似度去重（第二层，MinHash）
- 内容指纹去重（第三层，SimHash）
- 事件聚类（同一事件多篇报道聚合）

**验收标准：**
- [ ] URL 相同的文章自动去重
- [ ] 标题相似度 > 70% 的文章标记为重复
- [ ] 内容相似度 > 85% 的文章标记为重复
- [ ] 同一事件的多篇报道聚合展示
- [ ] 去重率 > 30%

---

**描述：** 用户注册、登录、个人资料管理

**功能点：**
- 邮箱注册/登录

- 密码重置

**验收标准：**
- [ ] 用户可通过邮箱注册
- [ ] 用户可通过 Google 账号登录
- [ ] 用户可通过 Apple 账号登录
- [ ] 登录状态保持 7 天
- [ ] 支持密码重置

---

#### F-05: 话题订阅

**描述：** 用户浏览和订阅感兴趣的话题

**功能点：**
- 话题分类目录
- 话题搜索
- 一键订阅/取消
- 订阅列表管理
- 推荐话题

**验收标准：**
- [ ] 用户可浏览分类目录
- [ ] 用户可通过关键词搜索话题
- [ ] 用户可一键订阅/取消
- [ ] 用户可管理订阅列表（排序、删除）
- [ ] 系统推荐相关话题

---

#### F-06: 信息流展示

**描述：** 展示用户订阅的最新信息

**功能点：**
- 综合信息流（所有订阅）
- 单话题视图
- 卡片式布局（标题 + 来源 + 时间 + 摘要）
- 下拉刷新
- 无限滚动分页

**验收标准：**
- [ ] 首页展示综合信息流
- [ ] 可切换到单话题视图
- [ ] 卡片展示标题、来源、时间、摘要
- [ ] 支持下拉刷新
- [ ] 信息流加载时间 < 2 秒

---

#### F-07: 搜索功能

**描述：** 搜索已聚合的内容

**功能点：**
- 全文搜索
- 搜索结果高亮
- 搜索历史
- 热门搜索

**验收标准：**
- [ ] 支持标题和内容搜索
- [ ] 搜索结果按相关性排序
- [ ] 搜索响应时间 < 500ms
- [ ] 保存搜索历史

---

#### F-08: 推送通知

**描述：** 智能推送重要更新

**功能点：**
- 重大更新即时推送
- 每日定时摘要推送
- 突发事件推送
- 推送频率控制（默认 3-5 条/天）
- 推送偏好设置

**验收标准：**
- [ ] 订阅话题有重大更新时推送
- [ ] 每天定时推送今日摘要
- [ ] 用户可设置每日推送上限
- [ ] 支持静默时段
- [ ] 推送打开率 > 15%

---

#### F-09: 书签收藏

**描述：** 收藏文章以便稍后阅读

**功能点：**
- 一键收藏
- 收藏列表
- 收藏分类
- 添加笔记

**验收标准：**
- [ ] 用户可一键收藏文章
- [ ] 用户可查看收藏列表
- [ ] 支持添加笔记
- [ ] 收藏同步到云端

---

#### F-10: 免费增值

**描述：** 免费和付费功能分层

**免费层：**
- 订阅话题上限：5 个
- 基础摘要
- 每日查看次数：20 次
- 有广告/品牌水印

**付费层（$4.99/月）：**
- 无限订阅
- 深度 AI 分析
- 优先推送
- 无广告
- 离线阅读

**验收标准：**
- [ ] 免费用户受限订阅数和查看次数
- [ ] 付费用户解锁全部功能
- [ ] 应用内购买流程顺畅
- [ ] 付费转化率基线建立

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 首页加载时间 | < 2 秒 |
| 搜索响应时间 | < 500ms |
| API 响应时间 P95 | < 500ms |
| 并发用户数 | 1000+ |
| 可用性 | 99.5% |

### 4.2 安全要求

| 要求 | 描述 |
|------|------|
| 数据加密 | 传输加密（HTTPS），存储加密（敏感数据） |
| 认证授权 | JWT + OAuth 2.0 |
| GDPR 合规 | 数据最小化、用户数据导出/删除 |
| 输入验证 | 防止 SQL 注入、XSS |
| API 安全 | 限流、认证、CORS |

### 4.3 合规要求

| 要求 | 描述 |
|------|------|
| 版权合规 | 摘要 + 链接跳转，不缓存全文 |
| GDPR | 隐私政策、数据处理协议、用户权利 |
| 热点数据采集 | 尊重 robots.txt、不突破技术防护、保留访问日志 |
| 应用商店 | 遵守 Apple/Google 审核指南 |

## 7. 发布计划

### 7.1 里程碑

| 阶段 | 目标 | 时间 | 交付物 |
|------|------|------|--------|
| Phase 1 | 后端核心 | 4 周 | API、采集管道、AI 摘要 |
| Phase 2 | 用户系统 | 3 周 | 注册登录、订阅管理 |
| Phase 3 | 移动应用 | 4 周 | Flutter iOS/Android |

| Phase 5 | 商业化 | 3 周 | 付费功能、应用上架 |

**总计：** 约 16 周（4 个月）

### 7.2 MVP 范围

**必须有：**
- RSS 内容采集
- AI 摘要生成
- 基础去重
- 用户注册登录
- 话题订阅
- 信息流展示

**可以没有：**
- 推送通知
- 每日简报
- 付费功能
- 高级搜索
